Sistema Operacional

O sistema operacional — programa mais importante do computador — consiste em um conjunto de softwares cuja função é administrar e gerenciar recursos de um sistema, desde componentes de hardware a programas de terceiros, estabelecendo uma conexão entre os recursos do computador e usuário.

Ele é o tradutor, o intérprete das nossas interações com a máquina, facilitando desta forma as ações e assimilações que se dão de ambos os lados.

Como funciona

* De forma genérica, entenda como um "computador" qualquer máquina de processamento automático de dados, como um desktop, notebook, celular, tablet ou até mesmo um console de videogame, por exemplo.

O sistema operacional (S.O.) é capaz de controlar todos os recursos do computador, como a unidade de processamento, a memória, os arquivos, os dispositivos de entrada e saída, entre outros; além de fornecer a base para o desenvolvimento dos programas de aplicação. Ele atua como um intermediário entre o usuário e o hardware de um computador, ou seja, é a interface entre os mesmos.

O S.O. introduz uma camada de abstração (tudo que não sabemos por baixo dos panos) entre o hardware e o usuário, que transforma comandos no mouse, teclado e solicitações do sistema, como gerenciamento de recursos (CPU, memória RAM), em linguagem de máquina, enviando instruções ao processador.

Este último os traduz para código binário, processa e executa os comandos, e envia as respostas como informações que aparecem na sua tela.

Muitas vezes o S.O. é considerado um software base, porque fornecece suporte às aplicações. Ele funciona como uma interface entre a aplicação e as rotinas de E/S com o hardware, ou seja, é preciso ter um sistema que faça essa conversa, e é nesse campo que o S.O. atua. Essa peça mais elementar de software opera em dois modos: o modo núcleo (modo supervisor) e o modo usuário.

No modo supervisor o acesso é completo a todo o hardware, e é possível enviar qualquer instrução que a máquina seja capaz de executar. O modo usuário é mais restrito para instruções que interferem no controle da máquina, mas é onde todo o resto do software opera.

Componentes

Um S.O. contém componentes divididos entre os para o usuário (como bibliotecas, programas e interface) e as instruções que compõem o seu núcleo (kernel) que permitem o funcionamento conjunto do computador.

Os componentes mais importantes são:

  • kernel: sua finalidade, de modo simples, é unir os recursos físicos e lógicos de um aparelho. Compõe a parte central do programa e responde por tarefas cruciais, como:

    • — Estabelecer a camada de abstração de baixo nível (linguagem de máquina) com o hardware;

    • — Gerenciar recursos como processador, RAM, sistemas de arquivos e dispositivos de entrada e saída (monitor, teclado, mouse, impressora, etc.);

    • — Gerenciar processos (execução) de programas;

    • — Gerenciar o uso de dispositivos, memória do sistema e chamadas dos programas, definindo quais têm prioridade.

  • segurança: abrange o processo de inicialização, as atualizações de software e a proteção dos recursos do sistema do computador, como CPU, memória, disco, programas de software e dados armazenados.

  • interface: é a tela de comandos apresentada por este programa, ou seja, a interface gráfica do software. Normalmente, nesta tela, existem várias imagens, ícones, campos de texto e demais ferramentas que auxiliam o utilizador a desempenhar suas tarefas no software.

!Diferença entre kernel e firmware

Ambos os conceitos confundem muitas pessoas.

Embora possam conversar entre si, são coisas diferentes. Um firmware ou software embarcado é um conjunto de instruções programadas diretamente no hardware, quem contém parâmetros específicos para a operação de um determinado dispositivo.

Exemplificando: em um S.O., a BIOS (Sistema Básico de Entrada e Saída) e UEFI (Interface Extensível Unificada de Firmware) são exemplos de firmware, pois comportam instruções voltadas para a operação do hardware de um computador.

A evolução dos sistemas operacionais

Hoje, o mercado está bem segmentado, mas muitas pessoas ainda usam o Windows. Contudo, a história nem sempre foi assim. Ao longo dos anos, diversas companhias desenvolveram sistemas para se adequarem a diferentes tipos de usuários.

Na primeira geração (1951-1959) os computadores ocupavam áreas enormes e os sistemas operativos não existiam. A tarefa dos S.O. era executada por técnicos, os quais ditavam o que seria realizado através do próprio hardware.

No início da computação surgiram os primeiros S.O. mas eram únicos, ou seja, cada computador tinha um S.O. único.

O primeiro S.O. que suportou centenas de utilizadores comuns foi o CTSS (Compatible Time-Sharing System - Sistema compatível de divisão por tempo) e serviu para a evolução até aos S.O. que existem hoje.

Tipos

  • Monotarefa: Permite a execução de um único programa por vez;

  • Multitarefa: Permite que mais de uma tarefa seja processada simultaneamente no sistema computacional (atualmente, a maioria dos S.O. é multitarefa);

  • Monoprocessados: Somente um processador pode ser gerenciado pelo S.O.;

  • Multiprocessados: Vários processadores podem ser gerenciados simultaneamente;

  • Monousuário: O S.O. só é capaz de suportar um único usuário acessando seus recursos.

  • Multiusuário: Vários usuários dividem os mesmos recursos do sistema computacional.
